# 🗡️ TRPG6: Console Multiplayer RPG

**TRPG6**는 C++로 개발된 멀티플레이어 텍스트 기반 RPG입니다. 단순한 터미널 게임을 넘어, 독자적인 렌더링 엔진, 멀티 프로세스 로깅 시스템, 그리고 실시간 네트워크 동기화를 통해 풍부한 게임 경험을 선사합니다.

---

## 🚀 주요 특징

### 1. Terminal Graphics

- **ASCII Art Engine:** `stb_image`를 활용하여 이미지를 컬러 ASCII 아트로 변환하고 렌더링합니다.
- **분할 UI 레이아웃:** `Top`(상태창), `CenterLeft/CenterRight`(본문 영역), `Bottom`(입력창)으로  나뉜 체계적인 화면 구성을 제공합니다.
- **동적 UI 알림:** 플레이 시간을 나타나는 `Timed UI` 시스템을 통해 사용자 경험을 개선했습니다.
- **ANSI 컬러 지원:** 무기 등급(Epic, Legendary 등) 및 상태를 색상으로 직관적으로 표현합니다.

### 2. Networking

- **실시간 동기화:** `NetworkManager`를 통한 서버-클라이언트 아키텍처로 멀티플레이어 환경을 구축했습니다.
- **협동 레이드 (COOP Raid):** 여러 플레이어가 참여하여 거대 보스를 공략하는 레이드 시스템을 지원합니다.
- **아레나 및 배팅:** 플레이어 간의 대전을 관람하고 아이템을 배팅할 수 있는 경쟁 시스템이 포함되어 있습니다.
- **통합 채팅:** 게임 플레이 중 언제든 `111` 명령어로 채팅 모드에 진입하여 소통할 수 있습니다.

### 3. Gameplay

- **직업 및 성장:** 탱커, 딜러, 힐러 등 역할군 기반의 스탯 보너스와 레벨업 시스템이 구현되어 있습니다.
- **아이템 및 경제:** 상점 이용은 물론, 플레이어 간 **아이템 및 골드 거래**가 가능합니다.
- **대장간:** 몬스터 전리품을 조합하여 무기를 제작하고, 무기를 합성하여 강화할 수 있습니다.
- **보스 패턴:** 단순 공격이 아닌 확률 기반의 특수 패턴을 가진 보스 AI가 전투의 긴장감을 더합니다.

### 4. System IPC

- **IPCManager:** Windows Named Pipes를 사용하여 메인 게임 외에 별도의 로깅 및 모니터링 창을 실행합니다.
    - `--logviewer`: 실시간 시스템 로그를 출력하는 전용 창
    - `--multiviewer`: 멀티 플레이 상태를 감시하는 전용 창

## 🏗️ 시스템 아키텍처

### **1. State Machine (상태 패턴)**

`IGameState` 인터페이스를 상속받아 게임의 각 흐름(전투, 상점, 아레나, 인벤토리 등)을 독립된 클래스로 관리합니다. 이는 `GameManager`에 의해 제어됩니다.

### 2. **Managers (싱글톤 패턴)**

주요 시스템은 싱글톤으로 설계되어 어디서든 쉽게 접근할 수 있습니다.

- `GameManager`: 전체 게임 상태 및 플레이어 데이터 관리
- `Renderer`: 터미널 출력 및 화면 버퍼 관리
- `NetworkManager`: 패킷 송수신 및 네트워크 세션 관리
- `TradeManager`: 거래 프로세스 및 유효성 검증 관리
- `IPCManager`: 프로세스 간 통신 및 자식 프로세스 생명주기 관리
- `ArenaNetworkManager`: 아레나 네트워크 프로세스 처리 및 패킷 관리

## 🛠️ 기술 스택

- **Language:** C++ 17/20
- Tool: Visual Studio 2022
- **Platform:** Windows (Win32 API 사용)
- **Networking:** Winsock2
- **IPC:** Named Pipes
- **Library:** `stb_image.h` (Image Processing)

---

## 🎮 게임 시작하기

### 사전 요구 사항

- Windows OS
- C++ 컴파일러 (Visual Studio 2022 권장)

### 빌드 및 실행

1. `TRPG6.sln` 파일을 Visual Studio에서 엽니다.
2. `x64` 구성을 선택하고 빌드합니다.
3. 실행 시 필요한 리소스 파일들이 `Resources/` 폴더 내에 위치해야 합니다.

---

## 📜 주요 조작법

- **숫자 입력:** 메뉴 선택 및 행동 결정
- **111:** 채팅 모드 전환 (다시 입력 시 명령 모드)
- **서버 실행:** 첫 번째 실행 시 서버 모드로 작동하며, 이후 클라이언트들이 접속할 수 있습니다.
